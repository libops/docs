These docs cover LibOps application templates, `sitectl`, `cloud-compose`, and the managed LibOps platform. They are for institutions that need a clear way to run open-source applications, whether they host those applications themselves or ask LibOps to manage the platform around them.

## Choose the level of support you need

LibOps starts with portable Docker Compose templates that can run on a laptop, a campus server, a single VM, or LibOps-managed infrastructure. Use the part that helps your team now, then add more support when the operational burden grows.

<CardGroup cols={4}>
  <Card title="Run the template yourself" icon="docker" href="/templates/app-support">
    Clone or fork a supported template and deploy it with Docker Compose on hardware you control.
  </Card>
  <Card title="Add sitectl" icon="terminal" href="https://sitectl.libops.io/install">
    Keep Compose underneath while adding contexts, app-aware commands, logs, rollouts, and service helpers.
  </Card>
  <Card title="Deploy with cloud-compose" icon="server" href="https://cc.libops.io/">
    Provision the VM, persistent volumes, and sitectl context for self-hosted deployments on supported clouds.
  </Card>
  <Card title="Use the managed platform" icon="cloud" href="/platform/managed-platform">
    Let LibOps handle hosting, firewall rules, Vault-backed secrets, preview environments, membership, and support.
  </Card>
</CardGroup>
