Open-source software provides essential infrastructure for GLAM institutions, colleges, and universities. These applications align with institutional values of openness and preservation — but running them at production scale has historically required a DevOps practice most institutions can't justify staffing.

LibOps exists to close that gap. We give institutions the technical foundation and operational patterns to run sophisticated open-source software confidently, so they can spend their effort on the mission instead of the plumbing.

## Preservation through standardization

Informal hosting often leads to institutional isolation. When a site is modified in place without a clear operational model, documentation drifts, upgrades become risky, and the application eventually separates from the upstream community.

<Warning>
The riskiest hosting pattern we encounter: a production site nobody on staff fully understands, customized in place, no path back to current upstream. By the time something breaks, the right answer is usually "rebuild from scratch."
</Warning>

LibOps encourages the opposite. Standardized, repository-backed deployment paths let institutions customize their applications without losing their connection to the shared project. Software stays maintainable, secure, and aligned with community standards over the long term.
